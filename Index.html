Definiciones de Elementos de Páginas Web

I. Orígenes de la Web

1. Web 2.0

Se refiere a la evolución de la web que permitió la interacción y colaboración entre usuarios, destacando la creación de contenido por parte de los mismos.

2. Web 3.0

Conocida como la web semántica, busca hacer que la información en la web sea más comprensible para las máquinas, mejorando la personalización y el uso de inteligencia artificial.

3. La W3C y los estándares

La World Wide Web Consortium (W3C) desarrolla estándares para asegurar el crecimiento a largo plazo de la web, garantizando que las páginas sean accesibles y funcionales.

II. Lenguajes de etiquetas

1. SGML

Standard Generalized Markup Language, un lenguaje de marcado que define un conjunto de reglas para crear lenguajes de marcado específicos.

2. XML

Extensible Markup Language, un lenguaje que permite estructurar y almacenar datos de manera legible para humanos y máquinas.

3. HTML

HyperText Markup Language, el lenguaje estándar para crear páginas web que define su estructura y contenido.

4. XHTML

Extensible HyperText Markup Language, una versión más rigurosa de HTML que combina la flexibilidad de HTML con la estructura de XML.

5. HTML5

La última versión de HTML, que introduce nuevas características como soporte para multimedia y mejoras en la conectividad.

III. Otros lenguajes relacionados al desarrollo web

1. CSS/CSS3

Cascading Style Sheets, un lenguaje de estilo para describir la presentación de un documento escrito en HTML o XML.

2. JavaScript

Un lenguaje de programación que permite la interactividad en las páginas web, creando efectos dinámicos.

3. Ajax

Técnica que permite la actualización de partes de una página web sin recargarla completamente.

4. Tecnologías y lenguajes del lado del servidor

Incluyen lenguajes como PHP, Python, Ruby y Node.js, que se ejecutan en el servidor y generan contenido dinámico.

5. Navegadores

Software que permite a los usuarios acceder a la web, interpretando y mostrando el contenido de las páginas.

6. Motores renderizados

Componentes de los navegadores que interpretan el código HTML, CSS y JavaScript para mostrar la página.

IV. Herramientas de desarrollo

1. Software WYSIWYG

What You See Is What You Get, editores que permiten crear contenido visualmente sin necesidad de escribir código.

2. Otras alternativas

  • Figma: Herramienta de diseño colaborativo.
  • InVision: Plataforma para el diseño y prototipado de experiencias digitales.
  • Zeplin: Herramienta que facilita la colaboración entre diseñadores y desarrolladores.
  • GitHub: Plataforma de desarrollo que permite el control de versiones.

3. Inspección de código desde el navegador

Herramienta que permite a los desarrolladores inspeccionar y modificar el código HTML y CSS en tiempo real.

V. El Diseño De Una Página Web

I. Tipos de páginas Web

  • Estáticas: Contenido fijo.
  • Dinámicas: Contenido cambiante según la interacción.
  • E-commerce: Para comercio en línea.
  • Blogs: Plataformas para compartir artículos.
  • Foros: Espacios para discusiones entre usuarios.

II. Puntos básicos en la elaboración de una página Web

  • Objetivo claro.
  • Diseño responsivo.
  • Usabilidad.
  • Accesibilidad.
  • Optimización SEO.

VI. Estructura De Una Página Web

I. Jerarquía de directorio de un proyecto web

Organización de archivos y carpetas para facilitar el acceso a los recursos del sitio.

II. Elementos de estructura para el diseño de una página Web

1. Regla para validar y certificar una página web

Utilizar validadores de código como el W3C Validator.

2. Reglas para escribir HTML

Mantener una estructura semántica y usar etiquetas correctamente.

3. Estructura básica de un archivo web

<!DOCTYPE html>
<html lang="es">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Título de la Página</title>
</head>
<body>
    <h1>Encabezado Principal</h1>
    <p>Contenido principal de la página.</p>
</body>
</html>
